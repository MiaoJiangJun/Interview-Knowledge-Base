# 1.FreeRTOS内存分配策略
heap_1：五种方式里面最简单的，但是申请的内存不允许释放：开连续数值，分配直接下移指针

heap_2：支持动态内存的申请和释放，但是不支持内存碎片的处理，并将其合并成一个大的内存块：采用最适应算法 (best fit)，将整个堆空间（ucHeap[] 数组）组织成一个按块大小升序排列的空闲块链表，如果找到的块比所需大很多（超过一个最小块大小），它会被拆分成两块：一块用于分配，剩下的块则重新放回空闲链表，释放时会将内存块直接重新插回空闲链表，但不会检查它与相邻的空闲块是否连续，也不会将它们合并成一个更大的块（拆分碎片）

heap_3：将编译器自带的 malloc 和 free 函数进行简单的封装，以支持线程安全，即支持多任务调用：它不自己管理内存池，而是简单地直接调用标准库的 malloc() 和 free()，并在此基础上添加了临界区保护。
~~~

; 栈空间大小设置
Stack_Size      EQU     0x00000400          ; 1 KB

; 堆空间大小设置
Heap_Size       EQU     0x00000200          ; 512 Bytes

                AREA    HEAP, NOINIT, READWRITE, ALIGN=3
__heap_base
Heap_Mem        SPACE   Heap_Size
__heap_limit
~~~

heap_4：支持动态内存的申请和释放，支持内存碎片处理，支持将动态内存设置在个固定的地址：在 heap_2 支持内存释放的基础上，增加了合并相邻空闲内存块的能力，首次适应算法。当释放一个内存块时，heap_4 会检查该块的地址前后是否有相邻的空闲块，将这些块合并成一个更大的空闲块。

heap_5：在 heap_4 的基础上支持将动态内存设置在不连续的区域上。heap_4初始化是自动使用一个大数组 ucHeap[] 作为唯一的连续堆。heap_5则需要你主动调用 vPortDefineHeapRegions() 函数，传入一个内存区域定义表，告诉它你有几块不连续的 RAM 以及它们的起止地址。
